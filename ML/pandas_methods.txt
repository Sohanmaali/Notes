# SECTION 1 — TOP-LEVEL pandas (`pd.*`) FUNCTIONS (ALL CORE ONES)

## Data Input

### `read_csv`

Reads data from a CSV file into a DataFrame.

```python
pd.read_csv("data.csv")
```

### `read_table`

Reads delimited text files (generalized CSV reader).

```python
pd.read_table("data.txt")
```

### `read_excel`

Reads Excel files into DataFrames.

```python
pd.read_excel("data.xlsx")
```

### `read_json`

Loads JSON data into a DataFrame.

```python
pd.read_json("data.json")
```

### `read_sql`

Reads SQL queries or tables into a DataFrame.

```python
pd.read_sql("SELECT * FROM users", conn)
```

### `read_html`

Parses HTML tables into DataFrames.

```python
pd.read_html(url)
```

### `read_parquet`

Reads Parquet columnar storage files.

```python
pd.read_parquet("data.parquet")
```

### `read_feather`

Reads Feather binary files.

```python
pd.read_feather("data.feather")
```

### `read_pickle`

Loads pickled Pandas objects.

```python
pd.read_pickle("data.pkl")
```

## Data Output

### `to_datetime`

Converts values to datetime.

```python

```

### `to_numeric`

Converts values to numeric types.

```python

```

### `to_timedelta`

Converts values to timedeltas.

```python

```

## Utility Functions

### `concat`

Concatenates multiple Pandas objects.

```python

```

### `merge`



PART 12 — GroupBy METHODS

(applies to both DataFrameGroupBy and SeriesGroupBy)

GroupBy enables the split → apply → combine workflow.
Nearly every serious analysis depends on this.

Core Group Operations (Foundational)
groupby(...)

Creates a GroupBy object (already done before calling methods).

df.groupby("category")

agg

Applies one or more aggregation functions per group.

df.groupby("cat").agg("mean")

apply

Applies an arbitrary function to each group.

df.groupby("cat").apply(lambda g: g.head(1))

transform

Applies a function and returns results aligned to original shape.

df.groupby("cat")["x"].transform("mean")


⚠️ Important truth:

apply = flexible, slow

agg = fast, summarized

transform = shape-preserving
